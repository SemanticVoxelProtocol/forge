# Documentation Boundary Policy

## Public User Docs
- README may link only to public user docs
- migration notes must be public if README references them

## Maintainer-Facing Public Docs
- release checklists and stable contributor policies may be tracked publicly

## Internal Development Docs
- docs under docs/superpowers/specs/ and docs/superpowers/plans/ are internal by default
- internal docs must not be required reading from README
