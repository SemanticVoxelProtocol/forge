# Migrating to forge schema 2.0.0

## What changed

Schema 2.0.0 makes file/function governance agent-auditable. Governed file and function manifests can now carry:

- `evidence` — source file/excerpt evidence used by the AI judgment
- `confidence` — `low`, `medium`, or `high`
- `assumptions` — explicit uncertainty or migration notes
- `needsHumanReview` — whether the agent must surface ambiguity
- function `observedSignature` / `contractSignature` split

This is a breaking schema change because old manifests did not record evidence, so forge cannot safely treat their language-specific governance claims as freshly verified.

## Automatic migration

When forge opens an existing `.svp/` project with schema `1.x`, it automatically runs the `1 -> 2` migration and updates `.svp/manifest.json` to `2.0.0`.

For existing file/function manifests, the migration is conservative:

- it does **not** invent source evidence;
- it sets `evidence: []`;
- it sets `confidence: "low"`;
- it adds a migration assumption explaining that evidence must be refreshed;
- it sets `needsHumanReview: true`;
- it preserves the legacy function `signature` and copies it to `contractSignature`;
- it leaves `observedSignature` empty unless it was already present;
- it bumps each migrated manifest revision and recomputes `contentHash`.

## What you may notice after upgrade

After migration, `forge check` may report governance evidence warnings for migrated file/function manifests. This is expected.

Run:

```bash
forge check --json
forge compile-plan --json
```

Then let your code agent handle the generated `review` tasks. The agent should inspect source files, refresh evidence hashes/excerpts, raise confidence only when justified, and clear `needsHumanReview` only when the ambiguity is resolved.

## When automatic migration can fail

Automatic migration can fail if the installed forge version does not include the required migration path, or if local `.svp/` files are damaged.

If forge tells you to upgrade the CLI, install a newer forge version and run the command again. If project data appears corrupted, restore from a known-good state and retry.
