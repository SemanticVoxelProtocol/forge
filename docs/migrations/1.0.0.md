# Migrating to forge 1.0.0

## What happens automatically

When you run forge against an existing `.svp/` project, forge attempts the supported migration path automatically.

Most users do not need to manage schema details by hand or run separate migration commands.

## What you may notice after upgrade

The first command that loads your project data may take an extra step while forge updates `.svp/` data to the supported format.

After that, normal commands should continue working with the upgraded project state.

## When automatic migration can fail

Automatic migration can fail if the installed forge version does not support the migration your project needs, or if local project data is incomplete or incompatible.

## What to do if it fails

Read the error message first.

If forge tells you to upgrade the CLI, install the newer forge version it requires and run the command again.

If the project data appears damaged or incomplete, restore the project from a known-good state and retry.

If you still cannot migrate, open an issue and include the forge version, the command you ran, and the full error message.
